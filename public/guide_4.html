Network Layer: Data Plane

The network layer's primary goal is to transport segments from a sending host to a receiving host. Unlike the transport layer (process-to-process) or the data link layer (hop-to-hop over a single link), the network layer is a network-wide concern, involving every host and router in the path.

1. Key Network Layer Functions

There are two vital functions performed by the network layer:

  • Forwarding (Data Plane): The local, per-router action of transferring a packet from an input link interface to the appropriate output link interface.
  • Routing (Control Plane): The network-wide process that determines the end-to-end path that packets take from source to destination.

1.1 The Analogy

  • Forwarding: Taking a specific interchange/exit while driving through a single city.
  • Routing: Planning the entire trip from New York to Los Angeles using a map.

2. Network Service Models

A service model defines the characteristics of end-to-end delivery of data between sending and receiving hosts.

2.1 The Internet's "Best Effort" Model

The Internet's network layer provides a best-effort service. This means:

  • No Guarantees: There is no guarantee that a datagram will eventually be delivered, that it will arrive in order, or that it will meet any specific timing/bandwidth constraints.
  • Why it works: Its simplicity has allowed for massive scalability and rapid deployment. Performance is often "good enough" due to over-provisioning of bandwidth and the use of Content Distribution Networks (CDNs).

3. Router Architecture

A router's primary job is to move packets from input links to the appropriate output links.

3.1 High-Level Components

  1. Routing Processor (Control Plane): Runs routing protocols (OSPF, BGP), maintains routing tables, and computes forwarding tables. Operates in the millisecond timeframe.
  2. Switching Fabric (Data Plane): Connects the router's input ports to its output ports. It is the "network inside the router." Operates in the nanosecond timeframe.
  3. Input Ports: Perform physical and link-layer functions, and decentralized lookup (using a copy of the forwarding table).
  4. Output Ports: Store packets received from the switching fabric and transmit them on the outgoing link.

3.2 Input Ports and Decentralized Switching

To avoid a central bottleneck, each input port has its own copy of the forwarding table.

  • Lookup: The port examines the destination IP and determines the output port.
  • Match plus Action: The "match" is the destination IP (using longest prefix matching), and the "action" is forwarding the packet into the switching fabric.
  • Queueing: If datagrams arrive at the input port faster than the switching fabric can process them, input queueing occurs. This can lead to Head-of-the-Line (HOL) Blocking, where a packet at the front of the queue prevents other packets behind it from moving forward, even if their output ports are free.

3.3 Switching Fabrics

The fabric is the heart of the router. There are three main types:

  1. Memory: Switching under direct control of the CPU. Packets are copied to system memory and then to the output port. (Limited by memory bandwidth).
  2. Bus: Input ports transfer packets directly to output ports via a shared bus. (Limited by bus speed).
  3. Crossbar (Interconnection Network): Uses a complex mesh of buses to allow multiple packets to be transferred simultaneously as long as they are going to different output ports. (High performance).

3.4 Output Ports and Buffer Management

If the switching fabric delivers packets faster than the output link can transmit them, output queueing occurs.

  • Buffer Management: Deciding which packets to store and which to drop when the buffer is full.
    • Tail Drop: Drop the arriving packet.
    • Priority Drop: Drop lower-priority packets.
    • RED (Random Early Detection): Randomly drop packets before the buffer is completely full to signal congestion to the sender (e.g., using ECN - Explicit Congestion Notification).

3.5 Packet Scheduling

Determining the order in which queued packets are transmitted:

  • FCFS / FIFO: First-Come, First-Served.
  • Priority Scheduling: Send highest priority packets first.
  • Round Robin: Cycle through different classes of traffic.
  • Weighted Fair Queueing (WFQ): A generalized version of round robin where each class is guaranteed a minimum percentage of the link's bandwidth.

4. Forwarding Logic: Longest Prefix Matching

When a router looks up a destination address in its forwarding table, it may find multiple matches (if prefixes overlap).

  • Rule: The router always uses the Longest Prefix Match—the entry with the most specific (longest) subnet mask that matches the destination IP.

[!TIP] Why Longest Prefix Match? This allows for hierarchical addressing. An ISP might have a broad rule for 200.23.0.0/16, but a specific organization within that ISP might have a rule for 200.23.18.0/23. The router will correctly send traffic for the specific organization to the more specific link.

5. IP Addressing (IPv4)

An IP address is a 32-bit identifier for a host or router interface. Every interface on the Internet must have a unique IP address (globally unique, or unique within its private network).

3.1 Dotted-Decimal Notation

Instead of reading 32 bits (e.g., 11011111 00000001 00000001 00000001), we use dotted-decimal format: 223.1.1.1. Each number represents one byte (8 bits), ranging from 0 to 255.

3.2 Subnets

A Subnet is a group of device interfaces that can physically reach each other without passing through an intervening router.

  • IP Structure: [Subnet Part] [Host Part]
  • Interfaces in the same subnet share the same high-order bits (the subnet part).

5.2 CIDR: Classless Inter-Domain Routing

CIDR is the modern standard for address allocation. It uses a Prefix notation: a.b.c.d/x, where x is the number of bits in the subnet portion of the address.

  • Example: 200.23.16.0/23 means the first 23 bits identify the subnet.
  • Route Aggregation: Hierarchical addressing allows an ISP to advertise a single large prefix (e.g., /20) to the world, even if it has divided that space into multiple smaller subnets (e.g., /23) for different organizations.

5.3 Practical Subnet Calculations

When calculating the capacity of a subnet:

  • Total Addresses: $2^{(32 - x)}$
  • Usable Host Addresses: $2^{(32 - x)} - 2$ (Subtract one for the network address and one for the broadcast address).
  • Example (/29): $32 - 29 = 3$ bits for hosts. $2^3 = 8$ total addresses. $8 - 2 = 6$ usable host addresses.

6. Getting an IP Address

6.1 ISP and ICANN Allocation

  • ISP Allocation: An ISP gets a block of addresses from ICANN (Internet Corporation for Assigned Names and Numbers) or regional registries.
  • Hierarchical Delegation: ICANN $\rightarrow$ Regional Registry $\rightarrow$ ISP $\rightarrow$ Organization.

6.2 DHCP: Dynamic Host Configuration Protocol

DHCP allows a host to dynamically obtain an IP address from a network server when it joins the network. This is known as "plug-and-play" networking.

The 4-Step DHCP Process (DORA)

  1. Discover: Client broadcasts "Is there a DHCP server?" (dest: 255.255.255.255).
  2. Offer: Server responds "I am a server; here is an IP you can use."
  3. Request: Client says "I'd like to use that IP."
  4. ACK: Server confirms "OK, you've got that IP."

Beyond the IP Address

DHCP typically provides four key pieces of information to a client:

  1. IP Address
  2. Subnet Mask
  3. Default Gateway (The address of the first-hop router).
  4. DNS Server (Name and IP address of the local DNS server).

5. Special IP Addresses

  • Network Address: Host portion is all 0s. Used to identify the subnet itself.
  • Directed Broadcast: Host portion is all 1s. Sent to all hosts on a specific subnet.
  • Local Broadcast: 255.255.255.255. Sent to all hosts on the local link.
  • Loopback Address: 127.0.0.1. Refers to the local machine (the "localhost").
  • Local Host: 0.0.0.0. Refers to "this computer" during startup (e.g., in DHCP Discover).

8. Link Layer Fundamentals (Integrated)

While typically a separate layer, the data plane heavily relies on Link Layer technologies to move packets across a single physical link.

8.1 MAC Addresses

A MAC (Media Access Control) address is a 48-bit permanent identifier assigned to a network interface (NIC).

  • Analogy: MAC is like a Social Security Number (permanent/portable); IP is like a Postal Address (location-dependent).
  • Notation: Hexadecimal (e.g., 1A-2F-BB-76-09-AD).

8.2 Ethernet

Ethernet is the dominant wired LAN technology. It is connectionless (no handshaking) and unreliable (no ACKs/NAKs; dropped frames are recovered only by higher layers like TCP).

  • Topology: Modern Ethernet uses a Switched topology (star).
  • Frame Structure:
    • Preamble (8 bytes): 7 bytes of 10101010 followed by 1 byte of 10101011 (Start of Frame - SoF). Used to synchronize clock rates.
    • Addresses: Destination and Source MAC (6 bytes each).
    • Type: Indicates the higher-layer protocol (mostly IP: 0x0800).
    • Data (Payload): The IP datagram.
    • CRC (4 bytes): Cyclic Redundancy Check for error detection. If an error is detected, the frame is dropped.

8.3 Ethernet Switches

A switch is a Link-Layer (Layer 2) device.

  • Plug-and-Play: Switches are transparent; hosts are unaware of their presence, and they require no manual configuration.
  • No MAC Address: Unlike routers, switch interfaces do not typically have MAC addresses assigned.
  • Self-Learning: Switches build a "Switch Table" automatically:
    1. Learning: When a frame arrives, the switch records the Source MAC address and the incoming port.
    2. Lookup: It examines the Destination MAC to decide the action.
  • Actions:
    • Forward: If the destination is in the table, send it to the specific port (Selective Forwarding).
    • Flood: If the destination is unknown or is a broadcast address (FF-FF...), send it to all ports except the incoming one.
    • Filter/Drop: Discard the frame if the destination is on the same segment it arrived from.
  • Collision-Free: Switches buffer packets and provide dedicated links to each host, eliminating collisions and isolating collision domains.

8.4 Switches vs. Routers

Both are store-and-forward devices with forwarding tables, but they operate at different layers.

Feature Ethernet Switch IP Router
Layer Link Layer (Layer 2) Network Layer (Layer 3)
Addressing MAC Addresses IP Addresses
Configuration Plug-and-Play (Transparent) Requires IP configuration
Logic Self-learning, Flooding Routing algorithms (OSPF, BGP)
Broadcasts Vulnerable to Broadcast Storms Provides protection against storms
Topology Confined to Spanning Tree Supports arbitrary topologies
Use Case Small networks (hundreds of hosts) Large networks (thousands of hosts)

8.5 Spanning Tree Protocol (STP)

While redundancy is good for reliability, it creates physical loops in a switched network.

  • The Problem: Broadcast Storms. Switches will multiply and forward broadcast frames forever in a loop, eventually crashing the network.
  • The Solution: Organizes switches into a loop-free Spanning Tree.
    • A subset of interfaces is disabled to break the loops while maintaining connectivity.
    • Invented by Radia Perlman (the "Mother of the Internet").

9. Address Resolution Protocol (ARP)

ARP is the "bridge" between the Network Layer (IP) and the Link Layer (MAC). It is used to find the MAC address of a host given its IP address.

9.1 How ARP Works

  1. ARP Table: Every host and router on a LAN maintains an ARP table (cache) that maps IP addresses to MAC addresses.
  2. Soft State: Information in the ARP table is "soft"—it times out (typically after 20 minutes) and is discarded if not refreshed.
  3. Plug-and-Play: Nodes create their tables automatically without administrator intervention.
  4. The Process:
    • A wants to send to B: If B's MAC is not in A's ARP table, A sends an ARP Query.
    • Broadcast: The query is sent to the broadcast MAC address FF-FF-FF-FF-FF-FF. All nodes on the LAN receive it.
    • Unicast Reply: B sees its IP in the query and sends an ARP Response directly to A (unicast) containing its MAC address.
    • Update: A updates its table and sends the data frame.

9.2 ARP Packet Format

ARP messages are encapsulated directly in Ethernet frames (Type 0x0806). Key fields include:

  • Hardware Type: e.g., Ethernet.
  • Protocol Type: e.g., IP.
  • HLEN & PLEN: Length of physical (MAC) and protocol (IP) addresses.
  • Operation: Indicates if the packet is a Request (1) or a Reply (2).
  • Sender/Target Addresses: Both MAC and IP addresses for the sender and target.

10. Routing Between Subnets (Addressing Walkthrough)

When sending a packet to a destination on a different subnet, the packet must pass through a router.

10.1 Key Principle: Subnet Determination

Before sending a packet, a host must determine if the destination is on the same LAN or a remote subnet:

  • The Logic: The host performs a bitwise AND operation between the destination IP and its own subnet mask to find the Network Prefix.
  • Binary Example:
    • Host IP: 153.169.4.235
    • Subnet Mask: 255.255.254.0 (Prefix: /23)
    • Expanding the 3rd byte: 4 = 00000100, 254 = 11111110.
    • 00000100 & 11111110 = 00000100 (which is 4).
    • Network Prefix: 153.169.4.0.
  • Result:
    • If the result matches its own network prefix, the destination is local (use ARP for the destination's MAC).
    • If they do not match, the destination is remote (send the frame to the Default Gateway MAC address).

10.2 IP vs. MAC Headers

  • IP Addresses: Source and Destination IP addresses NEVER CHANGE during the entire journey (from end-to-end).
  • MAC Addresses: Source and Destination MAC addresses CHANGE AT EVERY HOP (from link-to-link).

10.2 Step-by-Step Walkthrough (A $\rightarrow$ Router R $\rightarrow$ B)

  1. At Source A:
    • A realizes B is on a different subnet (using subnet mask).
    • A must send the packet to its Default Gateway (Router R).
    • IP Header: Source = A, Dest = B.
    • Ethernet Header: Source MAC = A, Dest MAC = Router R.
  2. At Router R:
    • R receives the frame, strips the Ethernet header, and looks at the IP destination (B).
    • R consults its forwarding table to find the correct outgoing interface.
    • IP Header: Source = A, Dest = B (Unchanged).
    • Ethernet Header: Source MAC = Router R (outgoing port), Dest MAC = B.
  3. At Destination B:
    • B receives the frame, strips the Ethernet header, and passes the IP datagram up the stack.

10.4 Prerequisites for Host Communication

For a host (A) to communicate with another host (B/C):

  • IP of Destination: Usually obtained via DNS (Domain Name System).
  • Subnet Membership: A determines if the destination is local using its own IP and Subnet Mask.
  • Default Gateway: A learns the IP of its first-hop router (R) usually via DHCP.
  • Gateway MAC: A finds the MAC address of R using ARP.

11. IP Fragmentation and Reassembly

Different physical networks have different MTU (Maximum Transmission Unit) limits (e.g., Ethernet is 1500 bytes). If a router receives a datagram larger than the next link's MTU, it must fragment it.

11.1 Fragmentation Mechanism

  • Identification (ID): All fragments from the same original datagram share the same ID.
  • Flags:
    • DF (Don't Fragment): If set to 1, the router drops the packet and sends an ICMP error instead of fragmenting.
    • MF (More Fragments): Set to 1 for all fragments except the last one (which is 0).
  • Fragment Offset: Indicates where in the original datagram this fragment's data belongs.
    • Unit: Measured in blocks of 8 bytes.
  • Constraints:
    • Payload Size: The payload size of every fragment (except the last) must be a multiple of 8.
    • Double Fragmentation: Fragments can be further fragmented by downstream routers if they encounter even smaller MTUs.

11.2 Reassembly

  • Where?: Reassembly is performed ONLY at the destination host to keep routers fast and simple.
  • Mechanism:
    • Timer: A reassembly timer is set when the first fragment arrives. If all fragments don't arrive before the timer expires, the datagram is discarded.
    • Missing Fragments: If one fragment is lost, the entire original datagram is discarded (IP does not recover missing fragments; it relies on higher layers like TCP).

12. ICMP (Internet Control Message Protocol)

ICMP is used by hosts and routers to communicate network-layer information, primarily for error reporting and diagnostics.

12.1 Detailed ICMP Types and Codes

ICMP messages are carried in IP datagrams (Protocol 1). They contain a type, code, and the first 8 bytes of the IP datagram that caused the error.

Type Code Description Example/Usage
0 0 Echo Reply ping response
3 0 Net Unreachable Router has no matching route entry
3 1 Host Unreachable Router has route, but ARP fails
3 3 Port Unreachable Used as traceroute stopping criteria
3 4 Datagram Too Big Fragment required but DF flag set
5 0/1 Redirect Redirect for Network/Host
8 0 Echo Request ping request
11 0 TTL Expired traceroute hop identification
12 0 Bad IP Header Header checksum or parameter error

12.2 Key Tools and Mechanisms

  • Ping: Tests reachability using Echo Request/Reply.
  • Traceroute:
    • Sends sets of 3 probes (UDP segments) with increasing TTL (1, 2, 3...).
    • Each router sends back an ICMP Time Exceeded (11/0) message.
    • Stopping Criteria: The destination receives the packet but can't find the UDP port, so it sends Port Unreachable (3/3).
  • Path MTU Discovery (RFC 1191):
    • Source sends packets with DF=1.
    • Routers unable to forward return Type 3 Code 4.
    • The "Unused" Field: The router includes the next-hop MTU in the low-order 16 bits of the ICMP header's "unused" field.

13. NAT (Network Address Translation)

NAT allows a local network to use a single public IP address for all its devices, helping mitigate IPv4 address exhaustion.

13.1 Private IP Ranges (RFC 1918)

These addresses are for local use only and are never routed on the public Internet:

  • 10.0.0.0 - 10.255.255.255 (10/8)
  • 172.16.0.0 - 172.31.255.255 (172.16/12)
  • 192.168.0.0 - 192.168.255.255 (192.168/16)

13.2 How NAT Works: Implementation

The NAT router maintains a Translation Table and performs three main steps:

  1. Outgoing: Replace the (Internal IP, Internal Port) with (NAT Public IP, New External Port). Update the translation table.
  2. External Storage: Remote servers respond using the NAT IP and the new port.
  3. Incoming: Look up the incoming destination port in the table to find the corresponding internal IP/Port and rewrite the header.

13.3 Pros and Cons

  • Advantages:
    • IPv4 Conservation: Use one public IP for many local hosts.
    • Flexibility: Change ISPs or local addresses without reconfiguring the entire network.
    • Security: Internal hosts are not directly addressable or visible to the outside world.
  • Controversies:
    • Layer Violation: Routers (Layer 3) should not modify Port numbers (Layer 4).
    • End-to-End Principle: Violates the concept that hosts should communicate directly without intermediate modification.
    • Traversal: Difficulty for P2P applications where external hosts need to initiate connections to hosts behind NAT.

14. Software Defined Networking (SDN) & Generalized Forwarding

Traditional routers use Destination-Based Forwarding (longest prefix match). Modern networks increasingly use Generalized Forwarding, where forwarding decisions are based on any header field (L2, L3, or L4).

14.1 The SDN Philosophy

  • Separation of Planes: The Control Plane (deciding where to send traffic) is logically centralized in a controller, while the Data Plane (executing the decisions) remains in the switches.
  • Programmability: Network administrators can program the behavior of switches through the controller using protocols like OpenFlow.

14.2 OpenFlow Abstraction: Match + Action

A switch contains a Flow Table. Each entry in the table consists of:

  1. Match (Pattern): A set of header values to match (e.g., Source IP, Dest Port, MAC Address). Wildcards (*) can be used.
  2. Actions: What to do with a matching packet:
    • Forward: Send to a specific output port.
    • Drop: Discard the packet (Firewall functionality).
    • Modify: Change header fields (NAT functionality).
    • Send to Controller: Encapsulate and send to the SDN controller for further processing.
  3. Stats: Counters for packets and bytes matched by the rule.

14.3 Unified Device Abstraction

The "Match + Action" model allows a single physical device to act as multiple types of middleboxes:

  • Router: Match: Destination IP Prefix $\rightarrow$ Action: Forward.
  • Switch: Match: Destination MAC $\rightarrow$ Action: Forward/Flood.
  • Firewall: Match: IP/Port $\rightarrow$ Action: Permit/Deny.
  • NAT: Match: IP/Port $\rightarrow$ Action: Rewrite Header.

14.4 SDN Controller (e.g., POX)

The controller is the "network operating system" that maintains global network state.

  • Logically Centralized: While it may be a distributed system for robustness, it provides a single consistent view of the network.
  • APIs:
    • Northbound API: Provides abstractions (like a network graph) to control applications (e.g., routing, firewall).
    • Southbound API: Protocols (like OpenFlow) used to communicate with and program the switches.
  • POX: A Python-based controller with built-in modules for hub behavior, L2 learning, and L3 learning.

14.5 OpenFlow Protocol Messages

OpenFlow defines how the controller and switches communicate over TCP.

Category Message Purpose
Controller-to-Switch Features Controller queries switch capabilities.
Modify-State Add, delete, or modify flow table entries.
Packet-Out Controller sends a packet out of a specific switch port.
Switch-to-Controller Packet-In Send a packet to the controller if no flow table match exists.
Flow-Removed Inform controller that a flow entry has expired.
Port-Status Inform controller of link/port changes (up/down).

14.6 Motivation: Why SDN?

Traditional destination-based routing has significant limitations in Traffic Engineering:

  1. Load Balancing: Traditional routing cannot easily split traffic to the same destination across multiple paths.
  2. Flow-Specific Routing: Routers cannot easily route traffic differently based on the source or transport protocol (e.g., send UDP on a low-latency path and TCP on a high-bandwidth path).
  3. Ease of Management: Centrally programming a network is less error-prone than configuring hundreds of individual routers running distributed protocols.

15. Data Center Networking

Modern data centers (like those of Google or Amazon) house hundreds of thousands of hosts and require specialized network architectures.

15.1 Hierarchical Architecture

  1. Blades: Individual servers mounted in racks.
  2. Top of Rack (TOR) Switch: A switch at the top of each rack connecting the blades (usually 40-100Gbps).
  3. Tier-2 / Tier-1 Switches: Connect multiple TOR switches together in a hierarchy.
  4. Border Routers: Connect the data center to the public Internet.

15.2 Key Features

  • Multipath: High-density interconnects between switches allow for multiple disjoint paths between any two racks, increasing both throughput and reliability.
  • Load Balancers: Application-layer devices that distribute incoming requests across many servers to prevent bottlenecks and ensure high availability.
  • Protocols: Innovations like RoCE (RDMA over Converged Ethernet) allow for direct memory access between servers with minimal CPU overhead. ECN (Explicit Congestion Notification) is often used in transport-layer congestion control (e.g., DCTCP).

16. Network Prototyping with Mininet

Mininet is a network emulator used to create virtual networks of hosts, switches, links, and controllers on a single machine.

16.1 Why Mininet?

  • Rapid Prototyping: Test complex topologies and SDN controllers (like POX) without needing physical hardware.
  • Real Code: Runs real Linux network software (e.g., web servers, iPerf, Ping).
  • Performance Modeling: Allows setting bandwidth limits and propagation delays on links.

16.2 Common Tools for Testing

  • Ping: Tests connectivity and latency.
  • iPerf: Measures maximum TCP and UDP bandwidth performance.

17. IPv6 (Internet Protocol Version 6)

IPv6 was developed to address the looming exhaustion of the IPv4 address space and to streamline packet processing.

17.1 Motivation

  • Address Space: 32-bit (IPv4) vs 128-bit (IPv6).
  • Efficiency: Fixed-length 40-byte header speeds up processing in routers.
  • QoS Support: Includes a "Flow Label" to identify and prioritize specific traffic flows.

17.2 IPv6 Header Fields

  • Version: 4 bits (set to 6).
  • Priority/Traffic Class: 8 bits; used for Quality of Service (QoS).
  • Flow Label: 20 bits; identifies packets in the same "flow."
  • Payload Length: 16 bits; size of the data following the header.
  • Next Header: 8 bits; identifies the upper-layer protocol (TCP/UDP) or extension headers.
  • Hop Limit: 8 bits (equivalent to IPv4 TTL).
  • Source/Destination Addresses: 128 bits each.

17.3 Key Differences: What's Missing?

To ensure high-speed forwarding, several IPv4 features were removed from the IPv6 header:

  • No Checksum: Removed to reduce processing time at every router (L2 and L4 layers already perform error detection).
  • No Fragmentation: Routers are not allowed to fragment IPv6 packets. If a packet is too large, the router drops it and sends an ICMPv6 "Packet Too Big" message.
    • Requirement: End hosts must perform Path MTU Discovery before sending data.
  • No Options: Options are moved to "Extension Headers" indicated by the "Next Header" field.

18. IPv6 Transition Strategies

Since the entire Internet cannot switch to IPv6 overnight, transition mechanisms allow the two protocols to coexist.

18.1 Dual Stack

A "Dual Stack" node has both IPv4 and IPv6 implementations. It can send and receive both types of datagrams, choosing the protocol based on the destination's capability (often determined via DNS).

18.2 Tunneling (IPv6 over IPv4)

Tunneling is used when two IPv6 nodes need to communicate but are separated by IPv4-only routers.

  • Encapsulation: The entire IPv6 datagram is placed inside the payload of an IPv4 datagram.
  • Addressing:
    • IPv6 Header: Source/Dest = Original IPv6 endpoints.
    • IPv4 Header: Source/Dest = The IPv4 addresses of the tunnel "entry" and "exit" points.
  • View: To the IPv4 routers, it looks like a normal IPv4 packet. To the IPv6 nodes, the IPv4 network looks like a single logical link (a "tunnel").

19. Error Detection and Correction

Errors can occur due to signal attenuation or noise. The Link Layer uses redundancy to detect and sometimes correct these errors.

19.1 Parity Checking

  • Single Bit Parity: Adds one bit to ensure the total number of 1s is even (or odd). Detects single-bit errors.
  • Two-Dimensional (2D) Bit Parity: Arranges data in a grid and calculates parity for every row and column.
    • Feature: Can detect and correct a single-bit error (the intersection of the row and column parity errors).

19.2 Cyclic Redundancy Check (CRC)

CRC is a powerful error-detection method used in Ethernet and WiFi.

  • The Math: Uses modulo-2 arithmetic (XOR).
  • Mechanism:
    1. Sender and receiver agree on an $r+1$ bit generator (G).
    2. Sender appends $r$ CRC bits (R) to the data D such that $<D, R>$ is exactly divisible by G (remainder 0).
    3. Receiver divides the incoming bits by G. If the remainder is non-zero, an error occurred.
  • Performance: CRC can detect all burst errors of length less than $r+1$ bits.

20. Multiple Access Protocols

When multiple nodes share a single broadcast channel (e.g., WiFi, traditional Ethernet), a protocol is needed to prevent/manage collisions.

20.1 Taxonomy of Protocols

  1. Channel Partitioning:
    • TDMA (Time Division): Each node gets a fixed time slot in each round. Unused slots go idle.
    • FDMA (Frequency Division): Each node is assigned a fixed frequency band.
  2. Random Access: Nodes transmit at the full rate whenever they have data. Collisions are allowed and recovered from.
    • ALOHA: Simple but inefficient.
    • CSMA: Listen before you talk.
  3. Taking Turns:
    • Polling: A master node invites other nodes to transmit.
    • Token Passing: A special token is passed; only the holder can transmit.

20.2 CSMA (Carrier Sense Multiple Access)

  • Basic Rule: "Listen before you talk." If the channel is busy, defer transmission.
  • Why Collisions Still Occur: Due to Propagation Delay. Two nodes may start transmitting nearly simultaneously because they haven't yet "heard" the other's signal.

21. CSMA/CD (Collision Detection)

CSMA/CD is used in wired Ethernet to reduce the time wasted by collisions.

21.1 The Algorithm

  1. Sense: If the channel is idle, start transmitting. If busy, wait.
  2. Transmit: Send the frame.
  3. Detect: While transmitting, listen for other signals.
  4. Abort: If a collision is detected, stop immediately and send a Jam Signal.
  5. Backoff: Wait a random amount of time (Binary Exponential Backoff) before retrying.

21.2 Binary Exponential Backoff

After the $m$-th consecutive collision for a frame:

  • The node chooses a random value K from the set ${0, 1, 2, \dots, 2^m - 1}$.
  • The node waits $K \times 512$ bit times before returning to Step 1.
  • Goal: Adapts to network load. More collisions $\rightarrow$ longer average wait time.

21.3 Minimum Frame Size (The 64-Byte Rule)

Ethernet requires a minimum frame size of 64 bytes (512 bits).

  • Why? To ensure the sender is still transmitting when a collision signal from the farthest point of the network returns. If the frame were too small, the sender might finish transmitting and think it was successful before the collision signal arrived.
