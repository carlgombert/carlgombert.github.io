Q9
HTTP/1.1 vs HTTP/2: What is one key technical improvement HTTP/2 makes and why does it matter?
Short Answer · HTTP
▾
Compare the two versions in terms of connection usage and latency.
Your answer
Key point: HTTP/2 introduces multiplexing — multiple requests and responses can be in-flight over a single TCP connection simultaneously. This eliminates head-of-line blocking at the HTTP layer, reducing page load times especially for many small objects. HTTP/2 also adds header compression (HPACK) and server push.
Q10
Iterative vs. recursive DNS resolution — which places more load on the local DNS server?
Short Answer · DNS
▾
Explain the difference and identify which approach makes the local DNS server do more work.
Your answer
Key point: In iterative resolution the local DNS server contacts each server in turn and assembles the answer itself — more load on the local server. In recursive resolution each queried server queries the next, offloading work to upstream servers.

Q17
How does P2P file distribution time scale vs. client-server as the number of peers grows?
Short Answer · P2P
▾
Your answer
Key point: In client-server, distribution time grows linearly with N. In P2P, each new peer also uploads to others — aggregate upload capacity scales with N, so P2P distribution time grows very slowly, making it dramatically more scalable.
Q18
True or False: A web cache can reduce response time even when the origin server's bandwidth is not the bottleneck.
True / False + Justify · Web Cache
▾
Justification
True. Even with fast origin bandwidth, a web cache serves content from a nearby proxy, eliminating wide-area propagation delay. The cache reduces the effective RTT, regardless of origin bandwidth.
Q19
Describe the full sequence from typing https://example.com to receiving the first byte of HTML (DNS, TCP, TLS).
Short Answer · Protocol Stack
▾
Your answer (numbered steps encouraged)
Key steps:
1. Browser checks local DNS cache; if miss, queries local DNS server.
2. Local DNS performs iterative lookup: root → TLD → authoritative → returns IP.
3. Browser initiates TCP 3-way handshake (SYN → SYN-ACK → ACK).
4. TLS handshake over established TCP (ClientHello → ServerHello → cert → key agreement → Finished).
5. Browser sends HTTP GET over the encrypted TLS tunnel.
6. Server responds; first byte arrives at client.

Q37
Key difference between link-state and distance vector in terms of information exchanged?
Short Answer · Routing
▾
Your answer
Link-state: Each router broadcasts its local link costs to all routers (flooding), so every router builds a complete topology map and runs Dijkstra's locally.

Distance vector: Each router shares its full distance vector only with direct neighbors, iterating via Bellman-Ford until convergence. No global topology is known.
Q38
Why does poisoned reverse fail to prevent loops involving three or more routers?
Short Answer · Count-to-Infinity
▾
Your answer
Poisoned reverse only poisons routes advertised back to the direct next-hop. A third router C (not the direct next-hop) may still advertise a stale low-cost path through A→B. B sees C's advertisement, updates its table to route through C, and the loop A→B→C→A forms — poisoned reverse never touched C's advertisement to B.

Wireless — 802.11
Q42
Why does 802.11 use CSMA/CA instead of CSMA/CD?
Short Answer · Wireless
▾
Your answer
Collision detection requires simultaneously listening while sending. In wireless, the transmitter's own signal is orders of magnitude stronger than any colliding incoming signal, making it physically impossible to detect a collision while transmitting. 802.11 avoids collisions before they happen using random backoff and optional RTS/CTS handshaking.
