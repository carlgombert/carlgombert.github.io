CSci 4211: Computer Networks - Master Study Reference

1. Network Layer: Control Plane Overview

The network layer is divided into two interacting parts: the Data Plane and the Control Plane. While the data plane handles the local, per-router function of forwarding packets from input to output ports, the control plane manages the network-wide logic that determines how packets are routed along the end-to-end path from source to destination.

1.1 Key Functions of the Network Layer

  • Addressing: Global identification of hosts and routers (e.g., IPv4/IPv6).
  • Forwarding (Data Plane): The process of moving a packet from a router’s input port to the appropriate output port.
  • Routing (Control Plane): The process of determining the end-to-end path taken by packets from source to destination.

1.2 Control Plane Approaches

There are two primary ways to implement the network control plane: 1. Per-Router Control (Traditional): Routing algorithms are implemented in every router. Each router contains a routing component that communicates with the routing components in other routers to compute forwarding tables. 2. Logically Centralized Control (SDN): A separate, remote controller (typically a server) computes and installs the forwarding tables in each router. This is the foundation of Software-Defined Networking (SDN). * Match-Action Rules: SDN routers use generalized forwarding. A table entry contains a Match (e.g., IP source/dest, Port, Protocol like TCP/UDP) and an Action (e.g., Forward to port \(X\), Drop, or Modify header). * Policy Control: Allows the controller to specify different paths for different types of traffic (e.g., route TCP traffic via one path and UDP via another).

2. Routing Paradigms and Graph Abstraction

2.1 Routing Goals

The primary goal of any routing protocol is to determine “good” paths through the network. * Path: A sequence of routers that a packet traverses from a given initial source host to the final destination host. * “Good” criteria: Typically defined as the path with the least cost, fastest speed, or least congested.

2.2 ICMP (Internet Control Message Protocol)

While the IP protocol handles the actual movement of data, ICMP is used for: * Error Reporting: Notifying the source when a packet cannot reach its destination or has been dropped. * Router Signaling: Communicating control information between routers and hosts. * Signaling Example: Used by tools like ping and traceroute.

2.3 Issues in Route Selection

Beyond finding the “least cost” path, several practical issues must be addressed by routing protocols: * Performance: Choosing routes based on specific metrics like smallest delay or maximum throughput. * Policy: Routes may be selected based on administrative rules (e.g., “do not route traffic through a competitor’s network” or “avoid .gov networks”). * Adaptability (Self-healing): The protocol must automatically respond to changes in topology (link failures) or network conditions with little to no human intervention. * Scalability: The algorithm must be able to support a massive number of hosts and routers without consuming excessive memory or bandwidth.

2.3 Classical Routing Paradigms

  1. Hop-by-Hop Routing (Default IP):
    • Each packet contains the destination address.
    • Each intermediate router independently chooses the next-hop for the packet.
    • Implication: Packets destined for the same host may take different paths if the topology or costs change between packets.
  2. Source Routing:
    • The sender specifies the entire path to the destination within the packet header.
    • Routers simply follow the path instructions provided by the sender.
    • Types: IP supports “loose” and “strict” source routing options.
    • Pros/Cons: Gives sender control, but fails if a specified link in the path goes down.
    • Examples: Virtual Circuit setups and MPLS (Multiprotocol Label Switching).

2.4 Graph Abstraction

To solve routing mathematically, networks are modeled as graphs \(G = (N, E)\): * Nodes (\(N\)): Represent routers. \(N = \{u, v, w, x, y, z\}\). * Edges (\(E\)): Represent physical links between routers. \(E = \{(u,v), (u,x), (v,x), \dots\}\). * Link Cost (\(c(x,y)\)): A value assigned to a link, which can represent: * Shortest Path: All costs set to 1. * Bandwidth: Costs inversely related to link capacity. * Congestion: Costs directly related to current traffic levels.

2.3 Classification of Routing Algorithms

  • Global (Centralized): All routers have complete topology and link cost info (e.g., Link State).
  • Decentralized: Iterative process where routers initially only know link costs to neighbors (e.g., Distance Vector).
  • Static: Routes change slowly over time.
  • Dynamic: Routes change quickly in response to topology or link cost updates.

2.4 Common Assumptions for Routing Algorithms

Both Link State (LS) and Distance Vector (DV) algorithms typically operate under the following assumptions: 1. Neighbor Discovery: Each router knows the address of its directly connected neighbors. 2. Cost Knowledge: Each router knows the exact cost of the link reaching each of its immediate neighbors. 3. Information Exchange: Global network information is discovered by nodes exchanging routing data with their neighbors.

Link State algorithms are centralized algorithms where every router must possess a complete map of the network topology and all link costs to compute paths.

LS vs. Distance Vector (DV) Information Exchange:

  • Link State (LS): Each node tells every other node (via broadcast) about its distance to its immediate neighbors.
  • Distance Vector (DV): Each node tells only its immediate neighbors its distance to every other node in the network.

3.1 Topology Dissemination

Before running the algorithm, nodes must share their local information: 1. Each router creates a Link State Packet (LSP) containing its ID, its neighbors’ IDs, and the associated link costs. 2. LSPs are distributed to all other routers in the network using controlled flooding. 3. Every router maintains a Link State Database, ensuring they all have an identical view of the network.

Loop-Free Guarantee: Routes computed by LS algorithms are guaranteed to be loop-free if: 1. Each router sees the exact same cost for every link. 2. Every router uses the same algorithm (e.g., Dijkstra) to compute the best path.

3.2 Dijkstra’s Algorithm

Dijkstra’s algorithm is an iterative process that computes the least-cost path from a single source node to all other nodes in the network.

Notation

  • \(c(x, y)\): Direct cost from node \(x\) to \(y\) (\(\infty\) if not neighbors).
  • \(D(v)\): Current estimate of the cost of the least-cost path from source to \(v\).
  • \(p(v)\): Predecessor node along the path from source to \(v\).
  • \(N'\): Set of nodes whose least-cost path is definitively known.

Algorithm Steps

  1. Initialization:
    • \(N' = \{\text{source node } u\}\)
    • For all other nodes \(v\):
      • If \(v\) is a neighbor of \(u\), \(D(v) = c(u, v)\) and \(p(v) = u\).
      • Else, \(D(v) = \infty\).
  2. Loop:
    • Find a node \(w\) not in \(N'\) such that \(D(w)\) is the minimum.
    • Add \(w\) to \(N'\).
    • Update \(D(v)\) for all neighbors \(v\) of \(w\) that are not in \(N'\):
      • \(D(v) = \min(D(v), D(w) + c(w, v))\)
  3. Termination: Stop when all nodes are included in \(N'\).

Post-Processing: Forwarding Table Construction

Once Dijkstra’s algorithm completes, the router has a least-cost-path tree. By tracing the predecessor nodes (\(p(v)\)) back to the source, the router identifies the first hop on the path to every destination. This first hop is then installed in the forwarding table.

  • Ties: If multiple paths have the same minimum cost, ties can be broken arbitrarily.

Algorithm Complexity

  • Time Complexity: \(O(n^2)\) comparisons for \(n\) nodes. In each of the \(n\) iterations, the algorithm must search all nodes not yet in \(N'\). With efficient implementations (e.g., using heaps), this can be reduced to \(O(n \log n)\).
  • Message Complexity: \(O(n^2)\). Each router must broadcast its link state information to \(n\) other routers. In a network of \(n\) nodes, each message crosses \(O(n)\) links, resulting in an overall complexity of \(O(n^2)\).

3.4 Potential Issues: Oscillations

If link costs are based on traffic volume (load-sensitive), route oscillations can occur. This is particularly problematic because: * Link costs are often directional and volume-dependent. * As traffic shifts to a “cheaper” path, that path becomes congested (more expensive), causing the algorithm to shift traffic back in the next iteration, leading to an unstable “ping-pong” effect.

4. Distance Vector (DV) Routing

Distance Vector algorithms are distributed, iterative, and asynchronous.

4.1 Basic Principles

  • Communication: A router tells its neighbors its estimated distance to every other router.
  • Bellman-Ford Equation: \(D_x(y) = \min_v \{ c(x,v) + D_v(y) \}\)

4.2 Link Cost Changes and Dynamics

  • “Good News Travels Fast”: If a link cost decreases, the information propagates quickly through the network as nodes immediately update their vectors and notify neighbors.
  • “Bad News Travels Slow”: If a link cost increases or a link fails, the network may suffer from the Count-to-Infinity problem.

4.2 Algorithmic Mechanics

The Distance Vector algorithm is characterized by three key properties: 1. Distributed: Each node receives information only from its directly attached neighbors, performs a calculation, and distributes the results back to its neighbors. 2. Iterative: The process continues until no more information is exchanged between neighbors. It is self-stopping—if a recomputation does not result in a change to the distance vector, the node does not send an update. 3. Asynchronous: Nodes do not need to operate in lock-step. A local iteration is triggered only by: * A local link cost change. * An update message (new distance vector) received from a neighbor.

4.3 Distance Tables vs. Distance Vectors

A critical distinction in DV routing is between the local storage and the advertised information: * Distance Table (\(D_x(Y, Z)\)): A table maintained by node \(x\) that stores the estimated cost to reach destination \(Y\) via neighbor \(Z\). * Rows represent all possible destinations in the network. * Columns represent each of node \(x\)’s immediate neighbors. * Distance Vector (\(D_x\)): The vector of the least-cost estimates from the distance table. Node \(x\) only advertises this vector (the “best” routes) to its neighbors, not the entire distance table.

4.4 State Information Diffusion

Information in a DV network propagates hop-by-hop over time: * At \(t=0\), a node only knows the cost to its immediate neighbors. * At \(t=1\), a node’s state information has reached its 1-hop neighbors. * At \(t=k\), the information has diffused to all nodes up to \(k\) hops away. * Convergence: Under stable conditions, the estimates \(D_x(y)\) will eventually converge to the actual least-cost path \(d_x(y)\) through this iterative diffusion.

4.5 The Count-to-Infinity Problem

This occurs when a node thinks it has a path through a neighbor, but that neighbor’s path actually went back through the first node. They will keep incrementing their costs indefinitely.

Solutions:

  • Split Horizon: A router never advertises a route back to the neighbor it learned that route from. For example, if \(X\) tells \(Y\) it has a path to \(Z\), \(Y\) will not advertise a path to \(Z\) back to \(X\).
  • Poisoned Reverse: A more aggressive version of split horizon. If \(Z\) routes through \(Y\) to reach \(X\), \(Z\) tells \(Y\) its distance to \(X\) is infinity. This explicitly prevents \(Y\) from ever trying to route back through \(Z\).
  • Limitations (The “Hack”): While these solutions solve “ping-pong” loops between two adjacent nodes, they do not solve loops involving three or more nodes. This is because a node might eventually hear about a “good” path to the destination from a third node, not realizing that the third node’s path actually goes back through the first node’s original (now broken) route.

4.6 Why Distance Vector is Still in Use

Despite its drawbacks (loops, slow convergence), DV remains relevant because: 1. Simplicity: It is easy to configure and implement. 2. Low Overhead: It requires significantly less CPU and memory compared to Link State algorithms. 3. Stability: In small or static networks where topology changes are rare, count-to-infinity is seldom an issue. 4. Legacy Support: Many older systems and home routers only support DV (specifically RIP). 5. Specialized Networks: DV variants like DSDV (Destination-Sequenced Distance Vector) are used in mobile ad-hoc and sensor networks.

5. Intra-AS Routing (IGP)

The Internet is divided into Autonomous Systems (AS) (administrative domains). Routing within an AS is called Intra-AS (or intra-domain) routing.

5.1 Common Intra-AS Protocols

  • RIP (Routing Information Protocol): A classic Distance Vector protocol (1980s).
  • OSPF (Open Shortest Path First): A classic Link State protocol; the current industry standard.
  • IS-IS: An ISO standard protocol, essentially identical in function to OSPF.
  • EIGRP (Enhanced Interior Gateway Routing Protocol): A DV-based protocol. Unlike RIP, it uses partial, triggered updates (only sending changes when they occur) rather than full periodic broadcasts, making it more efficient for large networks.

5.2 RIP (Routing Information Protocol)

  • Context: Commonly used in home routers and small business networks.
  • Algorithm: Distance Vector.
  • Metric: Hop count (max = 15 hops; 16 is infinity).
  • Exchanges: Every 30 seconds via Response Messages (advertisements).
  • Advertisement Limit: Each message can list up to 25 destination networks within the AS.
  • Failure Detection: If no advertisement is heard for 180 seconds, the link is declared dead, and the router invalidates routes via that neighbor.

5.3 OSPF (Open Shortest Path First)

  • Algorithm: Link State.
  • Transport: OSPF messages are sent directly over IP (rather than using TCP or UDP).
  • Features:
    • Open: Publicly available specification.
    • Multiple Metrics: Can use bandwidth, delay, etc., to calculate costs.
    • Security: All messages are authenticated to prevent malicious route injections.
    • Hierarchical Support: Can scale to large networks using areas.

5.4 Hierarchical OSPF

Hierarchical OSPF divides an AS into areas and a backbone. This restricts link-state flooding to specific areas, improving scalability.

Router Roles:

  1. Internal Routers: Flood link-state advertisements only within their local area and compute routing for that area.
  2. Area Border Routers (ABR): Sit at the edge of an area. They “summarize” the distances to destinations within their area and advertise them to the backbone.
  3. Backbone Routers: Run OSPF limited strictly to the backbone area.
  4. Boundary Routers: Connect the AS to other Autonomous Systems.

6. Inter-AS Routing: BGP (Border Gateway Protocol)

BGP is the de-facto inter-domain routing protocol, often called the “glue that holds the Internet together.”

6.1 BGP Sessions and Connections

  • eBGP (External BGP): Used to obtain subnet reachability information from neighboring ASes.
  • iBGP (Internal BGP): Used to propagate reachability information to all internal routers within an AS.
  • Gateway Router: A router at the “edge” of an AS that has links to routers in other ASes. Gateways perform both intra-domain and inter-domain routing. BGP-enabled border gateways are often referred to as BGP Speakers.
  • BGP Session: Two BGP routers (peers) exchange messages over a semi-permanent TCP connection.

6.2 BGP Message Types

BGP routers exchange four primary types of messages over their TCP connections: 1. OPEN: Opens a TCP connection to a remote BGP peer and authenticates the sending router. 2. UPDATE: The most critical message; used to advertise a new path to a destination prefix or to withdraw a previously advertised path. 3. KEEPALIVE: Sent periodically to maintain the connection in the absence of UPDATE messages; also used to acknowledge an OPEN request. 4. NOTIFICATION: Used to report errors in a previous message or to close a connection.

6.3 Path Attributes and Routes

A BGP route is defined as a Network Prefix + Attributes: * AS-PATH: The list of ASes through which the prefix advertisement has passed. Used to detect loops. * NEXT-HOP: The specific internal-AS router that should receive traffic destined for that prefix.

6.4 AS Relationships and Export Policies

  • Provider-Customer: The customer pays the provider for Internet access. Providers advertise all routes they know to their customers to provide global connectivity.
  • Peering: Two ASes (often of similar size) agree to exchange traffic for free.
    • Transit Rule: Peers provide transit only for their respective customers.
    • Restriction: A peer will not provide transit between its own peers or providers. For example, if AS A and AS B are peers, AS A will not carry traffic from AS B destined for AS A’s other peer AS C.
  • Export Summary:
    • To Customer: Advertise everything learned from anyone.
    • To Provider/Peer: Advertise ONLY your own routes and your customers’ routes.

6.5 BGP Policy Routing

BGP is fundamentally a Policy-Based protocol. A border gateway performs three main policy functions: 1. Import Filtering: Deciding whether to accept or decline a route advertisement from a neighbor (based on trust or path preference). 2. Route Selection: Ranking multiple available routes to the same destination. 3. Export Filtering: Deciding which routes to advertise to specific neighbors.

Common Policy Examples:

  • No-Transit Policy: Most ISPs only want to carry traffic that originates from or is destined for one of their own customers. They will not carry “transit” traffic between two of their providers or peers because it consumes resources without generating revenue.
  • Dual-Homed Customer: A customer connected to two different providers (e.g., for redundancy). To avoid becoming a transit provider itself, the customer will not advertise a route to Provider A through to Provider B.

6.6 Route Selection Process

If a router learns multiple paths to the same prefix, it selects the best route using: 1. Longest Prefix Match: First, find all available routes for the prefix. 2. Local Preference: A policy attribute set by the local admin (highest value wins). 3. AS Relationship (Rule of Thumb): Prefer routes in this order: Customer > Peer > Provider. This maximizes revenue and minimizes costs. 4. Shortest AS-PATH: Among routes with the same preference, choose the one with the fewest AS hops. 5. Hot Potato Routing: Choose the NEXT-HOP router that is “closest” in terms of intra-AS cost (e.g., lowest OSPF cost). The goal is to get the packet out of the local AS as quickly as possible, ignoring the inter-domain cost. 6. Tie-Breaker: If multiple paths remain, use a deterministic metric like the smallest BGP Router ID.

6.7 Why Separate Intra-AS and Inter-AS Routing?

The Internet uses different protocols for internal and external routing due to three main factors: * Policy: Inter-AS routing is dominated by policy (who pays whom, who is trusted). Intra-AS routing is under a single administrative control, so policy is less critical. * Scale: Hierarchical routing (Inter-AS) is necessary to keep routing tables manageable and reduce update traffic across the global Internet. * Performance: Intra-AS protocols (like OSPF) can focus on optimizing performance (e.g., shortest delay). In Inter-AS routing, policy almost always overrides performance metrics.

7. Wireless and Mobile Networks

7.1 Elements of a Wireless Network

  • Wireless Hosts: Devices running applications (laptops, smartphones, IoT). Wireless does not always mean mobility (e.g., a stationary wireless desktop).
  • Base Station: Typically connected to a wired network. Responsible for relaying packets (e.g., 4G/5G cell towers, WiFi APs).
  • Wireless Link: Connects hosts to base stations.
  • Radios: Modern devices often have multiple radios for different networks (e.g., iPhone 16 has ~11 radios for 5G, WiFi, Bluetooth, Satellite, etc.).
  • Network Modes:
    • Infrastructure Mode: Hosts communicate via a base station.
    • Ad Hoc Mode: Hosts communicate directly without a base station.

7.2 Wireless Link Characteristics

  • Decreased Signal Strength: Radio signals attenuate through matter (Path Loss).
  • Interference: Frequencies are shared by many devices.
  • Multipath Propagation: Signals reflect off objects, arriving at slightly different times.
  • SNR vs. BER: Larger SNR makes it easier to extract signal.
  • Dynamic Adaptation: Because SNR changes with mobility, the physical layer must dynamically adapt its modulation technique and transmission rate to maintain a low Bit Error Rate (BER).

7.3 Common Wireless Link Problems

  • Hidden Terminal Problem: A and C cannot hear each other but both transmit to B, causing a collision at B.
  • Exposed Terminal Problem: B hears C’s transmission and mistakenly thinks it cannot transmit to A.

Association and Scanning

Before a host can send data, it must associate with an Access Point (AP). * Passive Scanning: The AP periodically broadcasts Beacon Frames containing its SSID and MAC address. The host listens for these frames. * Active Scanning: The host broadcasts a Probe Request frame. APs in range respond with Probe Response frames.

MAC Protocol: CSMA/CA

Wireless uses Collision Avoidance rather than detection for several reasons: 1. Signal Disparity: The transmitted signal is much stronger than the received signal, making it difficult for the radio to detect a collision while transmitting. 2. Hidden Terminals: Even if a router could detect a local collision, it might miss one occurring at the receiver (hidden terminal problem). 3. Half-Duplex Nature: Most wireless NICs are half-duplex to conserve power and avoid self-interference, meaning they cannot transmit and receive simultaneously.

Protocol Mechanics: * DIFS (Distributed Inter-frame Spacing): Senders wait this long after sensing an idle channel before transmitting. * SIFS (Short Inter-frame Spacing): Receivers wait this short time before sending an ACK. DIFS > SIFS ensures that ACKs have priority over new data transmissions. * RTS/CTS: Small reservation packets containing a duration value. Other nodes hear this and update their NAV (Network Allocation Vector), a timer that tells them how long to remain silent. * Binary Exponential Backoff: If no ACK is received (indicating a collision or loss), the random backoff interval doubles for the next attempt#### 802.11 Frame Addressing An 802.11 frame contains 4 address fields, managed by To-AP and From-AP flags: * Address 1: The MAC address of the wireless host or AP intended to receive this specific frame (Receiver). * Address 2: The MAC address of the wireless host or AP transmitting this frame (Transmitter). * Address 3: The MAC address of the original source (if frame is coming from a router) or final destination (if frame is heading to a router). This is crucial for bridging the wireless BSS with the wired LAN. * Address 4: Used only in ad-hoc mode when frames are being forwarded between APs.

Case Study: Host to Router 1. Wireless Host sends frame: Addr1=AP_MAC, Addr2=Host_MAC, Addr3=Router_MAC. To-AP=1, From-AP=0. 2. AP receives it, strips 802.11 header, and creates an 802.3 (Ethernet) frame: Dest=Router_MAC, Source=Host_MAC.

Special Case: DHCP Discovery When a mobile device first joins and sends a DHCP Discovery: * Address 1: FF:FF:FF:FF:FF:FF (Broadcast). * Address 2: Device MAC. * Address 3: FF:FF:FF:FF:FF:FF (Original destination is everyone).

Crucial Note: IP addresses in the encapsulated datagram are never altered; only the MAC addresses in the 802.11/802.3 headers change as the packet moves through different link layers.

7.5 Cellular Networks (4G/5G)

Architectural Elements

  • User Equipment (UE): The mobile device.
  • RAN (Radio Access Network): Connects UEs to the base station (gNodeB in 5G).
  • Core Network: Situated between the RAN and the Internet.
    • CUPS: Control-Plane and User-Plane Separation.
  • Spectrum Bands (5G):
    • Low-band (<1 GHz): Long distance, lower speed (50-250 Mbps).
    • High-band (25-66 GHz / mmWave): Short distance (<1 mile), extremely high speed (up to 3 Gbps). Line-of-sight is critical as these signals have poor penetration.

5G Deployment Strategies

  • Non-Standalone (NSA): Supplement existing LTE infrastructure. The control plane runs over the LTE network, while the data plane uses 5G. This allows for faster deployment but doesn’t fully exploit 5G-native features.
  • Standalone (SA): No dependency on LTE. Uses a native 5G Core and 5G RAN. Requires heavier infrastructure investment but fully enables all 5G features (e.g., slicing, ultra-low latency).

mmWave Signal Characteristics

  • Obstructed By: Tinted/UV-coated glass, walls, and the human body.
  • Passes Through: Clear glass, backpacks, cardboard boxes.
  • Multipath: In outdoor environments, concrete buildings can effectively reflect mmWave signals, allowing connections to persist even if the direct line-of-sight is blocked (NLOS).

Resource Management

  • Resource Element (RE): The smallest unit of transmission, holding one OFDMA symbol on one subcarrier frequency for one time minislot.
  • Resource Block (RB): A bundle of neighboring (time, frequency) Resource Elements. For example, in 4G, an RB is 12 subcarriers by 7 minislots.
  • Scheduling: The RB is the smallest unit that can be scheduled for a specific source/destination.

5G Physical Channels

Data and control info are carried on specific physical channels: * Downlink (BS to UE): * PDSCH: Shared channel for user data. * PDCCH: Control channel for scheduling info. * PBCH: Broadcast channel for system info. * Uplink (UE to BS): * PUSCH: Shared channel for user data. * PUCCH: Control channel for radio feedback. * PRACH: Random access channel for initial connection.

RAN Protocol Stack (Processing Pipeline)

  1. RRC (Radio Resource Control): Manages the Control Plane. Handles connection setup, mobility, and coarse-grained policy decisions (scheduling priority, security). Does not process user data.
  2. SDAP (Service Data Adaptation Protocol): Maps IP flows to specific QoS (Quality of Service) levels.
  3. PDCP (Packet Data Convergence Protocol): Compresses/decompresses IP headers, handles ciphering, and provides integrity protection.
  4. RLC (Radio Link Control): Handles link-layer segmentation, reassembly, and reliable data transfer using ARQ (Automatic Repeat Request).
  5. MAC (Media Access Control): Manages buffering, multiplexing, and makes real-time scheduling decisions (typically on a 1ms loop).
  6. PHY (Physical Layer): Handles modulation, coding, and signal transmission.

Power Management: Coordinated DRX

To save battery, devices use DRX (Discontinuous Reception) cycles to sleep and wake up: 1. Active State: Radio is fully on for transmission/reception. 2. Inactivity Timer: After activity ends, the device waits for this timer. If it expires with no new data, the device enters sleep. 3. Short DRX Cycle: Periodic quick wake-ups to check for incoming traffic. 4. Long DRX Cycle: If no traffic is found in short cycles, the device enters a deeper sleep with longer intervals between check-ins to save more power. * Tradeoff: Longer sleep saves more energy but increases the latency for incoming packets (which must be buffered at the BS).

7.6 Mobility

Mobility can be categorized into a spectrum based on the network’s involvement: 1. No Mobility: Stationary device. 2. Low Mobility: Device moves between networks but powers down during the transition (e.g., closing a laptop in one building and opening it in another). 3. Mid Mobility: Device moves within the same provider’s WLAN or RAN. 4. High Mobility: Device moves across multiple provider networks while maintaining ongoing, seamless connections.

WiFi Mobility (ESS)

  • ESS (Extended Service Set): A collection of BSSs (Basic Service Sets) in the same layer-3 subnet, all sharing the same SSID.
  • Mechanism: When a mobile device moves between APs in an ESS, it can perform fast re-authentication to maintain its session at the link layer.

5G Handover Process

Handover occurs when a device (UE) moves its point of attachment from a Source BS (source gNodeB) to a Target BS. 1. Trigger: Source BS decides to handover based on signal strength or load balancing. 2. Preparation: Source BS sends a Handover Request to the Target BS. 3. Allocation: Target BS pre-allocates radio time slots and sends an ACK with connection info for the mobile device. 4. Execution: Source BS informs the UE of the change. The UE can now send/receive via the Target BS. 5. Forwarding: Source BS stops sending datagrams to the UE and instead forwards any incoming buffered datagrams to the Target BS. 6. Path Update: Target BS informs the AMF (Access and Mobility Management) and SMF (Session Management) that it is the new attachment point. 7. UPF Instruction: SMF instructs the UPF (User Plane Function) to change the tunnel endpoint to the new Target BS. 8. Completion: Target BS ACKs back to the Source BS; the handover is complete, and the Source BS can release its local resources. Traffic now flows through the new tunnel.

8. Integrated Network Operations: “A Day in the Life”

To understand how a network functions as a whole, consider a host (Host A) sending a packet to a server (Web Server W) through a campus LAN with switches and an AP.

8.1 Switch Self-Learning

Ethernet switches (like S1, S2, S3) maintain forwarding tables that map MAC addresses to specific physical ports. * Learning: When a switch receives a frame on port \(X\) from source MAC \(A\), it creates an entry: MAC A -> Port X. * Forwarding: When a frame arrives destined for MAC \(B\): * If MAC B is in the table, the switch forwards it to the specified port. * If MAC B is not in the table, the switch floods the frame (sends it out on all ports except the one it arrived on).

8.2 The AP as a Bridge

An Access Point acts as a link-layer bridge between the 802.11 wireless medium and the 802.3 wired Ethernet medium. * It maintains its own forwarding table to track which hosts are associated with its wireless interface vs. which are reachable via its wired Ethernet interface.

8.3 Synthesis Scenario: DNS Query

  1. Host B wants the IP for web.com. It generates a DNS query (UDP).
  2. Encapsulation: Host B creates an IP datagram (Dest=DNS_Server_IP) and an Ethernet frame (Dest=DNS_Server_MAC).
  3. Switching:
    • Switch S1 receives the frame, looks up DNS_Server_MAC, and forwards it toward Switch S2.
    • Switch S2 receives it and forwards it directly to the DNS Server \(N\) (assuming it has learned its port).
  4. Wireless Interaction: If the destination was a wireless laptop (Laptop L), Switch S2 would forward the frame to the AP. The AP would then translate the Ethernet frame into an 802.11 frame (Addr1=LaptopL_MAC, Addr2=AP_MAC, Addr3=Source_MAC).

9. Impact on Higher Layers

Wireless and mobility can negatively impact performance: * Packet Loss: Interpreted by TCP as congestion, leading to unnecessary window reduction. * Latency: Handover events and link-layer retransmissions increase delay, affecting real-time applications. * Bandwidth: A scarce resource compared to wired links.

10. Routing Algorithm Comparisons

10.1 Link State (LS) vs. Distance Vector (DV)

Feature Link State (LS) Distance Vector (DV)
Knowledge Global: Every router knows full topology. Local: Routers know neighbors only.
Convergence Fast, but may oscillate. Varies; slow with “bad news”.
Robustness Better: A router can only advertise its own links. Errors are isolated as each router computes its own table. Worse: A router can advertise incorrect path costs (e.g., black-holing). Errors propagate because routers use each other’s tables.
Complexity \(O(n^2)\) messages; \(O(n^2)\) compute. Exchange between neighbors; varies.

9.2 Ethernet (802.3) vs. WiFi (802.11)

Feature Ethernet (802.3) WiFi (802.11)
Duplex Full Duplex (Modern) Semi-Duplex (Half)
Multiple Access CSMA/CD (Collision Detection) CSMA/CA (Collision Avoidance)
Reliability High Low (Interference, Fading)
